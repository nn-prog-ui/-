1. このシステムは何をするの？

為替（FX）の価格を自動で分析して、「今買いやすいか・売りやすいか・どちらでもないか」を 画面に表示します。

表示される判定は3種類です：

判定意味
📈 買い候補 上がる可能性が高い条件が7つすべて揃っている状態
📉 売り候補 下がる可能性が高い条件が7つすべて揃っている状態
⏸ 見送り 条件が揃っていない・リスクが高い・どちらとも言えない状態
⚠️ 重要： このシステムは「参考情報の提示」だけを行います。 自動で注文を出す機能はありません。 最終的な売買の判断は、必ずあなた自身が行ってください。

対応している通貨ペア（8種類）

USD/JPY EUR/USD GBP/USD EUR/JPY AUD/JPY AUD/USD EUR/GBP GBP/JPY

2. 起動方法

1
ターミナル（黒い画面）を開く

Mac なら「ターミナル」、Windows なら「コマンドプロンプト」を開きます。

2
プロジェクトフォルダに移動してサーバーを起動する

以下のコマンドをコピーして実行してください。

pip3 install -r requirements.txt   # 初回のみ
python3 -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
3
ブラウザで開く

http://localhost:8000 にアクセスすると判定ページが表示されます。

サーバーを止めるには、ターミナルで Ctrl + C を押してください。

3. 毎日の使い方（3ステップ）

1
ブラウザで http://localhost:8000 を開く

ページを開くたびに最新データで分析が実行されます。

2
判定を確認する

画面の上部に大きく「買い候補」「売り候補」「見送り」のどれかが表示されます。

3
自分で判断して記録する

画面を最後まで読んで納得できたら「買い承認」「売り承認」「見送り記録」ボタンを押します。 ボタンを押しても実際の取引は発生しません。 判断の履歴が記録されるだけです。

4. 判定の読み方

判定は 7つの条件 をすべて満たしたときだけ出ます。

条件の満たし方判定結果
買い条件 7つ全部 ✅ 買い候補
売り条件 7つ全部 ✅ 売り候補
どちらか 6つ以下 見送り（6/7でも見送りになります）
6個揃っても「見送り」になるのは、安全を最優先にしているためです。 「惜しい！」と感じても、条件不足のときは慎重に判断してください。

5. スコアの読み方

スコアは −7 〜 +7 の数値で、 7つの条件のうち何個を満たしているかを表します。

スコア意味判定
+7 買い条件 7つすべて満たした 買い候補
+4 〜 +6 買い条件が多め・でも不足あり 見送り
0 前後 どちらともいえない 見送り
−4 〜 −6 売り条件が多め・でも不足あり 見送り
−7 売り条件 7つすべて満たした 売り候補

スコアバーの見方

画面では塗りつぶしたマス目でスコアを視覚的に表示しています。

例）買い 6/7
▲ 買い
6/7（あと1つで買い候補）
例）売り 2/7
▼ 売り
2/7（条件不足）

6. 各セクションの説明

📊 相場の流れ

項目意味
1日単位大きな流れ（長い目で見た方向）
4時間単位中くらいの流れ（半日〜1日の動き）
1時間単位直近の状態（最近の高値・安値を抜けたか）
上昇価格が上がる方向
下降価格が下がる方向
横ばいあまり動いていない

📈 価格分析の数値

項目意味目安
上がりすぎ・下がりすぎ（RSI） 0〜100の数値。過熱感を示す 70以上 = 買われすぎ、30以下 = 売られすぎ
値動きの激しさ 最近の値動きの大きさ 「通常」が安全。「高い」は注意
最近の高い価格 直近20時間の最高値 この価格を超えると「上昇突破」のサイン
最近の安い価格 直近20時間の最安値 この価格を下回ると「下降突破」のサイン

📉 価格の上限・下限（ボリンジャーバンド）

統計的に「このくらいの範囲で動くだろう」という上限・下限の帯です。

  • 上限に近い → 反落しやすい水準（上がりすぎ注意）
  • 下限に近い → 反発しやすい水準（下がりすぎ注意）

⚡ 上昇・下降の勢い（MACD）

短期と長期の平均価格の差を見て、今の勢いを判断します。

  • 上昇に転換 → 下から上に勢いが変わった
  • 下降に転換 → 上から下に勢いが変わった

🕯️ 価格パターン（直近の値動きの形）

ローソク足チャートの形から「上がりやすい形か、下がりやすい形か」を自動判定します。 ★の数が多いほど信頼性が高いとされます（あくまで参考）。

💰 取引の目安価格

項目意味
買う・売る価格の目安このあたりで取引を検討する価格
損失を止める価格ここまで逆に動いたら撤退する価格（損切りライン）
利益確定の目標価格ここまで動いたら利益を確定する価格
損失と利益の比率例）2.0 = 損失の2倍の利益を狙う設定

🌍 国際情勢（スコア補正）

世界情勢（戦争・経済制裁など）が USD に大きく影響するとき、スコアを ±1 補正します。 あくまで参考情報です。

🤖 AI コメント

分析結果を自然な文章で説明するコメントです。 右上に表示されるバッジで、どのAIが生成したか確認できます。

  • Claude AI — Anthropic の Claude（APIキー設定時）
  • OpenAI — ChatGPT（APIキー設定時）
  • ルールベース自動生成 — APIキーなしの場合

7. リアルタイムデータに切り替える

デフォルトでは CSV ファイルのデータを使います。 リアルタイムの価格で分析するには設定変更が必要です。

1
プロジェクトフォルダの .env ファイルを開く

テキストエディタで開いてください（メモ帳など）。

2
DATA_SOURCE=csv を書き換える 
DATA_SOURCE=yfinance
3
サーバーを再起動する

ターミナルで Ctrl + C で止めて、もう一度起動コマンドを実行します。

yfinance（Yahoo Finance）のデータは数分程度の遅延があります。 証券会社のリアルタイム価格とは少し異なる場合があります。

9. よくある質問

ボタンを押したら本当に注文されませんか？
はい、されません。データベースに「記録した」という履歴が保存されるだけです。 実際の取引とは一切連動していません。
スコア +6 でも「見送り」になるのはなぜ？
このシステムは安全を最優先して「全7条件が揃ったときだけ」を買い・売りと判定します。 6/7では「惜しい」ですが「見送り」です。
判定は何分ごとに更新されますか？
ページを開くたびに最新データで分析します。 自動スキャンは 設定ページ で間隔を変更できます（デフォルト60分）。
CSVファイルがなくても使えますか？
はい。DATA_SOURCE=yfinance に設定すれば Yahoo Finance からリアルタイムに取得できます。
AIコメントはどこから来ていますか？
ANTHROPIC_API_KEY を設定すると Claude AI、 OPENAI_API_KEY を設定すると OpenAI が使われます。 どちらもない場合はルールベースで自動生成します。 設定ページ で接続テストができます。
24時間自動で動きますか？
はい。サーバーを起動したまま放置すると、設定した間隔（デフォルト60分）で 自動スキャンが動き続けます。条件を満たしたときはメールで通知します。
スコアに「補正」と書いてあるのはなぜ？
世界情勢（戦争・経済制裁など）が USD に大きく影響するとき、スコアを±1 補正します。 「国際情勢」セクションに詳細が表示されます。あくまで参考です。

10. 安全に関する原則

1. 自動売買しない
ボタンを押しても注文は出ない。すべての売買はあなたが手動で行う。
2. 本番口座への接続機能はない
このシステムは FX 会社のシステムと接続していない。
3. APIキーはファイルに直接書かない
すべて .env ファイルで管理。コードに書いてはいけない。
4. 判断は人間が行う
AI・システムの判定はあくまで参考情報。最終判断は必ずあなた自身が行う。
